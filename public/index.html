Models and Generations

538astin Santy and Json Kim

The theme of this year's final project is "Explorable Explanation" and as I was preparing the template, I decided to build an explorable explanation myself. I've been wanting to explore approximation theory -- the math of representing complex objects with simpler, finite ones -- and this seemed like the right opportunity.

We encounter curves everywhere: from the outline of a leaf, to the arc of a bridge, the spiral of a shell, or the path of a satellite.

But computers do not store smooth, continuous curves. They store discrete numbers.

And so the question becomes:

  • How can a finite collection of numbers represent a smooth curve?
  • How close can we get?
  • What are the limits of that representation?

What does it mean to draw something smooth using only finitely many numbers?

Starting Simple

If a curve must be described using finitely many numbers, we need a family of functions controlled by finitely many parameters.

Polynomials are the simplest such family. We can fix a degree n, and the shape of the curve is completely determined by n + 1 coefficients.

Beyond their simplicity for computation, polynomials are smooth and easy to differentiate. More importantly, any sufficiently smooth function admits a Taylor polynomial expansion.

The gray curve is a target function (ex). The blue curve is a polynomial of the chosen degree. Increasing the degree enlarges the space of possibilities.

At first, this feels like freedom. The curve can bend, twist, flatten, or grow rapidly depending on those coefficients.

But the goal is not to create arbitrary polynomials.

It is to approximate curves that exist in the real-world.

Approximation

If the curve is already given, then choosing coefficients becomes a different kind of problem.

Among all polynomials of a fixed degree, which one comes closest?

That requires deciding what “close” even means.

As the degree increases, the error typically decreases. But unless the target is itself a polynomial, the error does not disappear entirely.

Increasing the degree enlarges the space of available curves.

One might expect the approximation to simply improve.

But something else begins to happen near the boundaries.

For certain functions, higher-degree polynomials start to oscillate. The fit improves in some regions and deteriorates in others.

More flexibility does not always mean more stability.

When High Degree Isn’t Enough

Increasing the degree enlarges the family of polynomials available to us.

It is tempting to think that more flexibility simply means better approximation.

But that intuition turns out to be incomplete.

Try interpolating the function 1 / (1 + 25x²) at equally spaced points.

For certain functions, as the degree increases, the approximation improves near the center, but begins to oscillate dramatically near the boundaries.

This behavior is known as Runge’s phenomenon.

The oscillations are not inevitable. Choosing interpolation points more carefully -- for example, clustering them near the endpoints as in Chebyshev nodes -- can reduce the instability.

But the polynomial remains global.

It improves in some regions and deteriorates in others because a single expression must accommodate the entire domain at once.

The issue is not smoothness, and not even degree.

It is the insistence on using one polynomial everywhere.

What if we allowed ourselves to use many smaller ones instead?

Enter Splines

Instead of increasing the degree indefinitely, we could divide the domain into smaller pieces and allow a different low-degree polynomial on each.

The curve would still be described by finitely many numbers. But now those numbers control local behavior rather than the entire domain.

Compare a single high-degree polynomial with a piecewise cubic spline.

Each segment is simple.

Smoothness is enforced where the pieces meet.

The representation is still finite-dimensional — but its structure has changed.

Flexibility now comes from adding segments, not from increasing a single global degree.

This is the idea behind spline methods, including Bézier curves and B-splines.

The oscillations disappear.

The approximation becomes stable.

It is tempting to think the problem is solved.

But the space is still finite.

What kinds of curves resist even this structure?

A Stress Test

Splines change the structure. They replace one global polynomial with many local ones.

The approximation becomes stable. And the oscillations disappear.

But the representation is still finite.

What happens if we test it against a curve whose structure persists at every scale?

Consider the logarithmic spiral — the kind we see with the Fibonacci sequence and the golden ratio.

As we add more segments, the spline approximation improves.

And yet, no matter how far we zoom, the spiral continues.

Its growth is scale-invariant.

The spline remains finite.

So although, stability is restored -- finiteness remains.

The Ambiguity of Finite Samples

There is also a consequence of finiteness.

Every time we commit to a finite set of sample points, we discard information about everything in between.

Given only finitely many measurements, infinitely many different curves pass through them exactly.

Two fundamentally different signals can produce identical values at every sample point.

A representation committed to those points alone cannot tell them apart.

This is what is known as aliasing.

Both curves pass through all N sample points. Given only the red dots, it is impossible to determine which signal produced them.

The polynomial interpolation we explored was not immune to this.

A degree n polynomial through n + 1 points is unique within its family but infinitely many other functions agree on those same points.

Runge's phenomenon from before was a form of aliasing.

Near the boundaries, the polynomial read a sparse set of sample values and inferred behavior that was not there, oscillating into frequencies the true curve never contained.

Splines narrowed the effect. Working locally, each segment has fewer points to misread. But each segment is still a cubic, and can only resolve variation at scales coarser than its width.

Adding more sample points raises the threshold but it cannot eliminate aliasing. It only pushes the ambiguity even more.

Finiteness doesn't merely limit how precisely we can approximate but it can also create ambiguity about what is being approximated at all.

From Curves to Computation

All of this may feel abstract.

But these choices — global vs local, finite vs infinite — appear every time a computer handles geometry or functions.

In CAD systems, smooth surfaces are represented by spline patches, then approximated again as triangle meshes for rendering and printing.

In neural networks, functions are approximated using finitely many parameters learned from data.

The mathematics changes form. The constraint does not.

Smooth structure, finite parameters.
